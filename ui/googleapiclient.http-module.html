Classes to encapsulate a single HTTP request.

The classes implement a command pattern, with every object supporting an execute() method that does the actual HTTP request.

_retry_request(http, num_retries, req_type, sleep, rand, uri, method, *args, >source code  Retries an HTTP request multiple times while handling errors. If after all retries the request still fails, last error is either returned as return value (for HTTP 5xx errors) or thrown (for ssl.SSLError). http: Http object to be used to execute request. num_retries: Maximum number of retries. req_type: Type of the request (used for logging retries). sleep, rand: Functions to sleep for random time between retries. uri: URI to be requested. method: HTTP method to be used. args, kwargs: Additional arguments passed to http.request. resp, content - Response from the http request (may be HTTP 5xx).
